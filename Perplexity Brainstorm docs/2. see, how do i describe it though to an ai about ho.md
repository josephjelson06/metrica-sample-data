What you want to tell the AI is **not** “build me a dashboard with some charts.”
What you want is:

**“Build a living, multimodal football intelligence assistant — not a static dashboard. The interface should stay continuously active, conversational, context-aware, and adaptive to the coach’s questions, with screen states that evolve naturally as the conversation deepens.”**[^1][^2][^3]

From your screenshots, the current UI looks more like a **query panel + result canvas** pattern, where each question produces a result card, but the system still feels page-like and command-driven rather than assistant-like.[^4][^5][^6]
You want it to feel more like the tablet is **present, listening, guiding, clarifying, and progressively revealing analysis**, not just executing one prompt at a time.[^7][^8]

## What to tell the AI

The clearest way to describe it is something like this:

> Build this as a voice-first, multimodal football intelligence assistant for coaches.
> Do not design it like a normal analytics dashboard with static panels and one-off outputs.
> The system should feel continuously active and conversational: always aware of current context, current team, current time window, current tactical thread, and prior questions.
> Each coach question should update the interface as a continuation of the same analysis session, not as a disconnected new page.
> The UI should adapt dynamically: clarify ambiguity, preserve context, suggest next questions, animate transitions between related states, and progressively reveal the right level of detail.
> The experience should feel like a smart analyst sitting beside the coach inside the tablet.[^3][^8][^7]

That is the **core idea** you are trying to express.

## The key distinction

The AI may currently be interpreting your request as:

- input box,
- suggested prompts,
- output chart,
- explanation box.[^5][^4]

But what you actually want is closer to this:

- persistent assistant presence,
- conversation-led navigation,
- stateful tactical memory,
- dynamic visual transitions,
- layered drill-down,
- proactive assistance,
- natural follow-up handling,
- adaptive UI depending on question type.[^9][^1][^7]

So the instruction to the AI should explicitly say:

**“This is not a prompt-response dashboard. It is a conversational operating interface.”**[^3][^9]

## What behavior you want

You want the assistant to behave like this:

- Always listening or ready-to-listen.
- Remembers what the coach just asked.
- Understands follow-ups like “now only first half,” “compare that,” “go deeper,” “show me why,” “only our left side.”
- Keeps the tactical thread alive across turns.
- Changes the screen in-place instead of constantly resetting views.
- Uses transitions that show continuity between states.
- Suggests useful next questions based on what is currently visible.
- Adapts the amount of explanation based on coach intent: quick answer, deeper explanation, or tactical diagnosis.[^8][^1][^7]

That “continuously active” feeling is the most important part.

## Better product framing

Use language like this when prompting the AI:

### Short version

> Design this as an always-on football coaching assistant, not a dashboard. It should feel like an intelligent analyst embedded in a tablet, continuously tracking context and guiding the coach through a live analytical conversation.[^1][^3]

### Stronger version

> The interface should behave like a multimodal tactical copilot. Voice is the primary interaction mode, touch is secondary, and the visual system should morph according to the coach’s intent. Every response should preserve conversational context, maintain analytical continuity, and offer relevant next-step interactions. Avoid static dashboard layouts and disconnected result pages.[^9][^1][^7]

## Design principles to explicitly tell the AI

You should give the AI principles, not just features.

### 1. Assistant-first

> The product should feel like talking to a football analyst, not operating BI software.[^10][^3]

### 2. Context persistence

> Every new question should inherit the current team, phase, minute window, selected players, and tactical focus unless the coach changes them explicitly.[^8][^7]

### 3. Progressive disclosure

> Show only the amount of information needed for the current question, then let the coach go deeper naturally.[^11][^12]

### 4. In-place evolution

> The screen should evolve from the current state instead of replacing everything abruptly with a new screen. Use transitions to preserve continuity.[^13][^8]

### 5. Conversational repair

> If the request is vague, ask a short football-aware clarifying question instead of failing.[^14][^15]

### 6. Proactive guidance

> After answering, the system should suggest the most likely next tactical questions.[^10][^7]

### 7. Mixed modality

> Voice should lead, visuals should explain, touch should refine.[^2][^1]

## The exact kind of prompt you can give

Here’s a prompt you can directly give to an AI builder:

> Build the football system as a voice-first multimodal assistant, not a static dashboard.
> The interface should feel alive, continuously active, and conversational.
> It should behave like an intelligent football analyst embedded in a tablet for a coach.

> The UI should adapt dynamically based on the question type: orientation questions, passing questions, structure questions, transition questions, chance-creation questions, and player questions should all produce different screen behaviors.
> Use smooth, meaningful transitions so each answer feels like a continuation of the previous one.
> The assistant should remain present at all times with a listening state, context ribbon, evolving explanation panel, and suggested next questions.
> Do not build this as a traditional left-sidebar analytics app with static result cards. Build it as a conversational tactical intelligence interface.[^1][^7][^8]
